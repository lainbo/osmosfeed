    什么是知识蒸馏？它是如何实现的？ ChatGPT 作为通用大模型，在各个行业它都算得上半个专家，但是让它跑起来的成本实在是太高了，在很多场景下，我们只需要用到它在某个子领域的一小块知识，那如何保证在推理效果不变的情况下，获得一个类似的小模型呢？https://arxiv.org/pdf/1503.02531.pdf，这篇论文详细介绍了一项技术，叫做“知识蒸馏”，论文是 2015 年发表的，这方面的实践已经十分成熟了。 知识蒸馏是一种基于 Teacher-Student Network 的模型压缩技术，将知识从较大的模型中分离出来，赋予给较小的模型，从而获得一个单领域认知更强、部署成本更低、推理速度更快的压缩模型。知识蒸馏的关键是将大模型的泛化能力传授给小模型，所谓的泛化能力，就是能够在给定输入的时候，可以符合预期地给出输出结果。 在进行模型训练的过程中，我们会准备一些硬标签，也就是已经分好类、打好标的输入集，将硬标签传给大模型后，它会输出一些包含不同概率的结果，例如手写数字“2”，它会识别出“2 的概率为 95%”、“3 的概率为 2%”，“7 的概率为 1%”等等，这些结果集称之为 soft target（分布概率，如图一），这个结果集的信息量相比硬标签更加丰富，如果让 Student 来学习这些知识，就可以得到一个更小的模型了。 这里的问题在于，soft target 的熵太低了（所谓的熵太低，就是说信息的有序性比较高），“2 的概率为 95%”基本上就可以断定结果是 2，而模型不仅仅要去学习什么是 2，同时也要去学习什么不是 2，那如何降低 soft target 的熵呢？论文引入了温度（T）的概念，通过将 soft target 中的概率分布除以温度来降低其熵（如图二），这种操作使得概率分布更平滑，减少了标签之间的差距，可以帮助小模型更好地学习大模型的知识。 更详细的讲解，可以阅读这篇文章：https://zhuanlan.zhihu.com/p/102038521，温度越高，信息熵就越高，此时子模型对负标签内容的感知就会越深刻，实验中一般会把 T 控制在 1~20 左右。 另外，还找到了一个开源项目 torchdistill，https://github.com/yoshitomo-matsubara/torchdistill，它把各个顶会和期刊上提出的关于知识蒸馏的论文都做了一次工程实现，你甚至都不需要写代码，通过简单地配置 yaml 就可以实现大模型知识向小模型的转移。  ( 2 min )
