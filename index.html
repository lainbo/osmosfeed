    经常看到社区推出各种 LLM，有 7B、13B、60B 等等，这些数值代表着模型的参数量，大家都知道数值越大，意味着模型的规模越大，那这些参数量到底是如何被计算出来的呢？ ChatGPT 中的 GPT 全称是 Generative Pre-trained Transformer，它是基于 Transformer 架构建设的，市面上的大模型基本也采用此架构，它们都是四层设计，分别是： 1）Embedding 层：用于将输入的标记（token）映射到高维的词嵌入空间，参数的数量取决于词汇表的大小和词嵌入的维度 2）Transformer 层：每个 Transformer 层包括多个子层，例如自注意力层（self-attention layer）和前馈神经网络层（feedforward neural network layer）；参数的数量取决于每个子层的权重和偏置参数数量，以及模型中的 Transformer 层数 3）Layer Normalization：归一化层通常用于规范模型中间层的输出，这些层的参数数量取决于归一化层的输入维度 4）Output 层：大多数语言模型在最后添加一个用于分类的线性层，用于将模型的输出映射到分类标签 大模型的参数量就是上面四层的参数总量之和。为了一探究竟，我在 Hugging Face 上找了个稍小的模型，dolly-v2-3b，它是 Databricks 创建的一个 28 亿参数 LLM ，源自 EleutherAI 的 Pythia-2.8b，使用 model.named_parameters() 可以打印模型中的所有参数（如图一），可以清晰看到每一层的参数量，将其累加： 128716800 + (2560 * 4 + 19660800 + 7680 + 6553600 + 2560 + 26214400 * 2 + 10240 + 2560) * 32 + 2560 * 2 + 128716800 = 2775086080 结果是 27.7 亿参数，四舍五入就是 28 亿。这种计算方式非常精确，也有更快捷的方式估算出大模型的参数总量，看到一篇文章，https://zhuanlan.zhihu.com/p/624740065，提供了一个估算的方法（如图二），由于 Transformer 这一层的模式比较固定，而且参数量最大，因此仅计算这一层即可，文章推到出可以使用 12 * L * H^2 作为公式来进行估算，其中 L 为模型层数，H 为隐藏层维度。 dolly-v2-3b 的层数是 32，隐藏层维度是 7680（如图三），按照公式计算出来的结果是 22 亿，相差不大。  ( 2 min )

    【打工人系列】做好信息同步
    准备新增一个系列，记录自己作为打工人，在职场中摸爬滚打的经验教训，权当作为自己的总结，如果能提供任何小小的帮助，就算产生额外的价值了。 在最近的工作中，需要作为技术负责的角色，跟进一个大型项目。在这个过程中，又更深地意识到了自己在信息同步这方面的缺陷。因此，也想系统总结一下自己遇到过的教训和思考。 信息同步的重要性 有人可能会有一些想法：「我最后把事情做好不就够了，还需要做这些吗？」，「平时交流已经够多了，我还需要专门整理汇总吗？」，「项目已经有 PMO 了，还需要我来做信息同步吗？」 答案是肯定的：做好信息同步非常重要。 对上，信息同步是有效管理老板预期、证明自己工作把控能力的重要手段。  老板把一个任务交给你来跟进，肯定不希望这个事情从此变成了一个黑盒。预期的 deadline 是什么时候？有什么风险吗？人力投入情况如何？这些情况老板一概不知，如何能放心交给你来负责呢？你在过程中如何证明你对这个任务有足够的把控能力呢？做好信息同步，不要再让老板去猜，去等结束那天开结果的盲盒了。 对内，信息同步是推动项目成功、获得同僚肯定、赢得职场信用的有效方法。  职场不可能没有合作，信息差越大，合作的阻力就越大。项目每天都有新的进展和问题，信息千头万绪，如何能保证所有参与者有一致的目标？如何保证所有交付节点能够顺利完成？若能够主动整理和汇总关键信息，让项目的每一个进展都如期发生，最终保证项目的交付，自然也能够积累起自己的职场信用。 对自己，信息同步是帮助自己理清思路，把事情做得更快更好的一种技巧。  信息同步不是瞎同步，而一定是你付出精力梳理现状、挖掘潜在问题、思考整理后的结果。这个过程对理清楚自己的思路，更好地开展后面的工作大有裨益。 如何做好信息同步 拿我来说，其实很多时候不是不知道信息同步的重要性。但在工作中，往往会后知后觉自己没有做好这一步。这里通过我的实践，提供几个踩过的坑： 跳出当前角色思考。  作为打工人，很容易陷入工作惯性，只专注于自己手头的工作，忽视了其他的事情。每天保证留出几个时间，暂停手头的工作，站在其他的视角问自己几个问题：如果我是老板，我对这个项目的进度和结果放心吗？如果我是 PMO，我觉得项目流程中有哪些问题？如果我是合作方，我们的合作流程清晰明确吗？ 从不同角度多想一步，自然会更容易发现工作和项目中遗漏的重要信息。 注意信息同步的内容和形式。  往往会看到有人所做的信息同步，就是在群里发一条消息、喊一嗓子，跟平时打招呼和闲聊别无二致。但这条消息的信息量是否完备？表达是否清晰？重点是否突出？这些都是需要经过审慎思考的。这绝对不是形式主义，而是减少误解发生的可能、同时突出职业性和专业性的重要表现。 「同步了」不等于「同步到了」。  信息同步不是把消息往项目群里发一下就可以了。在前几天的项目中，每天晨会都会产生一大堆的 todo 和结论，同时伴随着若干条重要的信息，例如人力的安排、排期的变更等。这些都被一股脑地发到了项目群里，然后被迅速地淹没在了消息的海洋中。 重要的信息，确保它们的确被派发到需要关注的角色了，最好确认得到了反馈和回复 ，这是很容易忽视的一点。 把信息同步当成自己一项真正的工作对待 。我经常觉得自己会做好信息同步，但每天的工作很容易让我忘记一些细节。最后等对方问起的时候，才懊悔自己遗漏了这一点。这本质上还是没有把信息同步当做自己十分重要的工作来看待。应该在什么时间，把什么信息同步给谁，这个事情值得被记为一个带 deadline 的 todo 项。 报喜报忧同样重要 。不止是同步关键问题，也需要同步关键进展和结论，越重要的项目越是应当如此。

    美食是一场发现之旅
    重口味的老家菜 ⬆️ 在老家的味道中，凝结着时间和情感的纽带。这些味道扎根于每个人心中，不仅仅是食物的滋味，更是土地赋予我们深刻的印记。然而，大部分时候，我们却往往被这些印记所困住。 在味觉面前，我们常常画地自牢。 但这并非批判，而是一种视角的转换。如果你偏爱某一地方的菜系，那并没有错，更不值得批判。 然而，我一直无法理解为何美食的地域性能够如此强烈地区分不同的省份，甚至在味蕾上产生极强的边界。为何人与人之间的生理差异微乎其微，而美食的地域性却如此明显呢？我们经常听到这样的说法：“我是四川人，无法接受清淡的广东菜”、“我是广东人，吃不了重庆火锅”、“我是湖南人，华北地区都是美食荒漠”。 我不认为美食的地域性是一种隔阂，相反，它是一种特殊的文化，让我们有更多机会去探索和理解这个多元味道的世界，还有它演进的历史。 作为湖北人，我对家乡重油重盐重辣的美味始终有着难以割舍的情感，每次回家都是我首要满足的美味诉求。然而，一旦离开家乡，我发现不同地域的人群也有着各自的美味，但我却像困在一个地方，过往的印记让我很难解开这个味蕾的束缚。 接受粤菜成为了一个转折点。一开始，我无法接受其清淡甚至带着淡淡微甜的味道，直到在兼职的粤菜饭店饿了一两天，突然之间身心都可以接受了。自此，心境突然开阔了，对食谱的上限和下限也不再有了底线，从重盐重辣，到清淡原味，我基本都可以接受。 就在这个月，我还去了无锡一趟，这是与常州并称的甜食之都，甚至小笼包都是甜的。然而，当我真正去了当地，尝试了一些本地菜肴之后，发现那种甜并不是我想象中的厌恶，甚至最终，我有些能够理解其中独特的风味。 而一旦味蕾打开，我可以毫无顾忌，品尝印度的咖喱、埃塞俄比亚的英吉拉、越南的米粉、韩国的泡菜、日本的料理，以及贝加尔湖畔的腌制鱼，甚至在英格兰的炸鱼薯条，我都可以品出其中的有趣味道。我发现，原来人可以不再是那个只会在家乡味道中徘徊的人。我开始理解，芥末原本只是一种压制腥味的调料，但在不同的料理中却变成了一种全新的味道体验，食物本身也在变化。 埃塞俄比亚菜：英吉拉 ⬆️ 就如对粤菜的转变一样，我们的味蕾是可以改变的，但这需要抵抗本能的，突破之后，我们可以体验更多。 我觉得，味蕾就像是兴趣，但兴趣不应该是一种感觉，而是一种过程，因为人不会天生就喜欢所有东西，需要一种缘分，或者引路人。
